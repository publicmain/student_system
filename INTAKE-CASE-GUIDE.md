# 入学案例详情页 — 功能说明文档

## 1. 页面概述

入学案例详情页是整个入学管理流程的**唯一操作中心**。一个学生从注册到入学的所有工作——材料收集、申请文件生成、签证办理、费用管理、文件收发、到校跟进——全部在这一个页面完成。

页面采用**双栏卡片式布局**，每个功能模块是一张独立卡片。卡片可以拖拽排序、折叠、隐藏，布局自动保存到本地。

---

## 2. 页面顶部结构

```
┌──────────────────────────────────────────────────────────────┐
│  ← 返回列表    学生姓名 · 项目名称              [状态切换]   │
├──────────────────────────────────────────────────────────────┤
│  ⓘ 流程时间线                                               │
│  ①已注册 → ②收集材料中 → ③合同已签 → ④签证办理中 →         │
│  ⑤IPA已收 → ⑥已付款 → ⑦已到校 → ⑧已入学 → ⑨已关闭         │
├──────────────────────────────────────────────────────────────┤
│  标签栏：ⓘ 概况 │ 📄 申请表文件 │ 🛂 签证状态 │ ...          │
└──────────────────────────────────────────────────────────────┘
```

**流程时间线**：9 个阶段节点，当前阶段高亮。点击节点可切换案例状态。

### 📦 文件总览 (`all_files`)

**位置**：左栏顶部（最醒目位置）
**权限**：principal, intake_staff, counselor 可见 | student_admin 不可见
**显示条件**：始终显示

**职能**：
汇聚该学生入学流程中产生的**所有文件**，一目了然。不需要去各个模块分别找文件。

**文件来源（5 类）**：

| 分类 | 颜色标识 | 来源 | 示例 |
|------|---------|------|------|
| 🔴 申请文件 | 红色 | 系统自动生成的 PDF | Student Application Form, Form 16, V36 |
| 🔵 代理上传材料 | 蓝色 | 代理通过 agent.html 上传 | 护照首页、成绩单、在读证明 |
| 🟣 文件收发（发出）| 紫色 | Staff 发给学生的文件 | Offer Letter、签证材料清单 |
| 🟢 学生回传 | 绿色 | 学生通过链接回传的文件 | 签名合同、补充材料 |
| 🟠 案例文件 | 橙色 | Staff 在案例中上传的内部文件 | 审核记录、内部备注 |

**交互**：
- 按分类分组，每组显示文件数
- 每个文件显示名称、大小、下载按钮
- 代理上传材料显示审核状态（已通过/已退回/待审核）
- 按日期倒序排列（最新在前）
- 顶部显示文件总数和总大小

---

### 📋 材料收集与申请文件 (`mat_adm`)

**位置**：右栏顶部
**权限**：principal, intake_staff, counselor 可见
**显示条件**：始终显示

**职能**：
这是入学申请的**核心操作区**。从发起材料收集到生成正式申请文件，全流程在此完成。

**三个区块**：

#### 区块 1：材料收集

```
┌─────────────────────────────────────────┐
│ 📁 材料收集                    [PENDING] │
│ 已审核 2/5 项           1 待审核         │
│ ████████░░░░░ 40%                       │
│                                          │
│ 护照首页    必须   ✓ 已通过     📥       │
│ 在读证明    必须   待审核    ✓ ✗ 📥      │
│ 成绩单      必须   待上传                │
│ 银行存款证明       待上传                │
│ 语言成绩证明       待上传                │
│                                          │
│ 🔗 Agent 填写链接        [复制链接]      │
└─────────────────────────────────────────┘
```

**操作**：
- **发起材料收集**：选择中介公司和联系人 → 系统自动创建请求 + 生成 Magic Link + 发送邮件
- **审核材料**：对每个已上传的材料项点击 ✓（通过）或 ✗（退回）
- **复制链接**：把 Agent 填写链接发给代理
- **进度条**：实时显示审核进度

#### 区块 2：学生信息表 (UIF)

```
┌─────────────────────────────────────────┐
│ 👤 学生信息表 (UIF)          [已提交]    │
│                                          │
│ [生成 3 份申请文件 (SAF/Form16/V36)]    │
└─────────────────────────────────────────┘
```

**状态流转**：
- **草稿中**：代理正在填写
- **已提交，待处理**：代理已提交，可以生成文件
- **已处理**：已生成过申请文件

**操作**：
- 点击"生成申请文件"→ 系统从 UIF 数据创建 ADM Profile → Python 生成 3 份 PDF
- 生成过程有轮询提示（2 秒检查一次，最多 30 秒）
- 生成完成后自动刷新页面

#### 区块 3：申请文件

```
┌─────────────────────────────────────────┐
│ ✅ 申请文件 (3 份)          [全部就绪]   │
│                                          │
│ 📄 Student Application Form  708KB [下载]│
│ 📄 Form 16 (Student Pass)    435KB [下载]│
│ 📄 V36 (Additional Info)     215KB [下载]│
└─────────────────────────────────────────┘
```

**三份 PDF 说明**：
| 文件 | 用途 | 页数 |
|------|------|------|
| **Student Application Form (SAF)** | 学校自己的入学申请表 | 3 页 (A4) |
| **Form 16** | ICA 学生准证申请表 | 2 页 (Letter) |
| **V36** | 学生准证附加信息表 | 2 页 (Letter) |

### 🛂 签证状态 (`visa`)

**位置**：左栏
**权限**：principal, intake_staff 可见 | student_admin 不可见
**显示条件**：案例进入"合同已签"阶段后解锁，或已有签证记录

**职能**：
跟踪学生的 Student Pass / 签证申请进度。

**信息**：
- 签证状态（未开始/申请中/已获批/已拒签）
- 申请日期、获批日期
- IPA Letter 信息
- SOLAR 申请号

---

### ✅ 任务清单 (`tasks`)

**位置**：左栏
**权限**：所有角色可见
**显示条件**：始终显示

**职能**：
管理该入学案例的待办事项。

**功能**：
- 添加任务（标题、截止日期、优先级、分类）
- 标记完成/未完成
- 按截止日期排序
- 学管老师只看到"入学"和"回访"类任务

---

### 📁 材料清单 (`materials`)

**位置**：左栏
**权限**：principal, intake_staff, counselor 可见
**显示条件**：案例进入"收集材料"阶段后解锁

**职能**：
管理该学生需要提交的入学材料（与材料收集模块的区别：这里是内部管理视角，材料收集是对外代理协作视角）。

---

### 🔀 文件收发中心 (`file_center`)

**位置**：右栏
**权限**：仅 principal, intake_staff 可见
**显示条件**：始终显示

**职能**：
与学生/家长进行文件的双向收发。

**操作**：
- **新建**：上传文件 + 填写标题 → 可选"立即发送"或"保存草稿"
- **发送**：通过邮件把文件链接发给学生
- **要求回传**：勾选后学生可通过链接上传回复文件
- **催办**：对逾期未回传的记录发送催办邮件
- **关闭**：标记收发完成

**筛选标签**：
- 全部 | 已发出 | 已回传 | 待回传 | 已逾期 | 已完成

**每条记录的操作（三个点菜单）**：
- 发送给学生 / 重新发送
- 编辑
- 催办
- 复制链接
- 下载文件
- 下载回传件
- 标记完成
- 删除

---

### ✈️ 到校信息 (`arrival`)

**位置**：左栏
**权限**：所有角色可见 | student_admin 不可见
**显示条件**：案例进入"IPA 已收"阶段后解锁

**职能**：
记录学生到达新加坡的信息。

**信息**：
- 预计到达日期
- 实际到达日期
- 航班信息
- 接机安排
- 住宿信息

---

### 📊 满意度调查 (`survey`)

**位置**：右栏
**权限**：仅 student_admin 可见
**显示条件**：案例进入"已入学"阶段后解锁

**职能**：
学管老师在学生入学后发送满意度调查并查看结果。

## 4. 案例状态流转

```
①已注册 registered
  ↓
②收集材料中 collecting_docs     ← 发起材料收集、代理填表
  ↓
③合同已签 contract_signed       ← 解锁签证状态卡片
  ↓
④签证办理中 visa_in_progress    ← 提交签证申请
  ↓
⑤IPA已收 ipa_received           ← 解锁到校信息卡片
  ↓
⑥已付款 paid                    ← 财务确认
  ↓
⑦已到校 arrived                 ← 记录到校信息
  ↓
⑧已入学 oriented                ← 解锁满意度调查、移交学管
  ↓
⑨已关闭 closed                  ← 案例归档
```

**状态切换**：点击流程时间线上的节点可前进/回退状态。

## 6. 卡片操作

每张卡片右上角有 4 个按钮：

| 按钮 | 功能 |
|------|------|
| `←` / `→` | 移动到左栏/右栏 |
| `∧` / `∨` | 折叠/展开 |
| `×` | 隐藏卡片（可通过"重置布局"恢复） |
| `⠿` (拖拽手柄) | 拖拽排序 |

布局变更自动保存到浏览器本地存储，下次打开同一案例时恢复。

## 7. 核心业务流程

### 流程 A：代理提交 → 生成申请文件

```
Staff 在入学案例中点击"发起材料收集"
  ↓ 选择中介公司和联系人
  ↓ 系统创建请求 + 生成 Magic Link + 发送邮件
  ↓
代理收到邮件，打开 Agent 填写链接
  ↓ 12 步表单：个人信息、家庭、教育、工作、财务、签名...
  ↓ 上传护照、成绩单等材料
  ↓ 电子签名
  ↓ 提交
  ↓
Staff 在入学案例中看到：
  - 材料收集进度更新
  - UIF 状态变为"已提交"
  ↓ 审核各项材料（通过/退回）
  ↓ 点击"生成 3 份申请文件"
  ↓
系统自动：
  - 从 UIF 数据创建 ADM Profile
  - 写入家庭、教育、工作、签名等子表
  - Python 在原 PDF 模板上叠加数据
  - 生成 SAF (3页) + Form 16 (2页) + V36 (2页)
  ↓
Staff 在入学案例中下载 3 份 PDF
所有文件同时出现在"文件总览"中
```

### 流程 B：文件收发

```
Staff 在"文件收发中心"点击"新建"
  ↓ 上传文件 + 填写标题
  ↓ 可选：勾选"要求学生回传" + 设截止日期
  ↓ 可选：勾选"立即发送"
  ↓
学生收到邮件，打开链接查看/下载文件
  ↓ 如果要求回传：学生上传回复文件
  ↓
Staff 看到状态变更：已查看 → 已回传
  ↓ 下载回传件
  ↓ 标记完成
```

## 8. 技术说明

- **卡片系统**：基于 `_CASE_CARD_META` 配置，通过 `_buildCard()` 函数动态渲染
- **布局持久化**：`localStorage` 按 case ID 存储，键名 `case_card_layout_{caseId}`
- **数据加载**：单次 API 调用 `GET /api/intake-cases/:id` 返回所有卡片数据
- **实时刷新**：部分操作（审核、生成文件）完成后自动调用 `renderIntakeCaseDetail()` 刷新
- **PDF 生成**：异步 fire-and-forget，前端轮询检查状态（2s 间隔，最多 15 次）
- **文件存储**：所有文件保存在 `uploads/` 目录，通过 UUID 文件名访问
